### 范例 2：什么是 Hermes 协议（认知类）

**内容目标**：让读者了解 Hermes 协议是什么，能跟人讲清楚

**v5 prompt**：

```
内容目标：让读者了解 Hermes 协议是什么，能跟人讲清楚

内容：
Hermes 协议：AI 与外部工具 / 数据源交互的标准化协议。

核心定义：
- AI 助手调用外部工具的统一接口
- 解决"AI 如何安全高效使用工具"的问题
- 相当于 AI 时代的 USB 标准

3 个核心机制：
1. 消息路由：AI 发出指令 → 路由到正确工具
2. 工具调用：标准化格式 → 工具执行 → 返回结果
3. 状态管理：跟踪多次调用的上下文

适用场景：
- 复杂 AI Agent
- 多步任务自动化
- 需要外部数据的 AI 应用

参考图：纯白 3D 渲染风格（科技类参考图）

呈现约束：
- 核心定义"AI 时代的 USB 标准"用大字号居中
- 3 个核心机制卡片化（2x2 或竖排）
- 整体简洁、信息密度中等
```

```
✓ 长文本 / 推理能力强 · 安全性高 · 写代码顶级
```

## 五、通用内容结构速查

> 不同内容类型有不同的"标准结构"。内容完整 → 读者能 follow；内容残缺 → 读者卡壳。
> 这里是常用类型的"4 要素清单"，写内容时按这个查。

### 步骤类（操作流程 / How-to）

| 要素 | 角色 | 示例 |
|---|---|---|
| **步骤编号** | 锚定顺序 | Step 1 / 1️⃣ / ① |
| **步骤标题** | 动词 + 对象 | "安装 Claude Code" |
| **具体操作** | 命令 / 工具 / 操作 | `npm i -g ...` |
| **关键提示** | 注意事项 | "需要 Node 18+" |

**4 要素齐全 → 内容完整。少一个 → 读者卡壳。**
