# 小红书图文生图 Prompt 生成器

## 设计哲学

**目标是内容，不是图。**

skill 的一切都为"内容被高质量呈现"服务。

- 图是手段，内容是目的
- 风格是手段，传达是目的
- 排版是手段，重点突出是目的

**agent 角色 = 内容目标守护者。**
- 锁定：内容目标是什么？读者看完要记住什么 / 感受什么 / 做什么？
- 守护：一切呈现（风格 / 排版 / 视觉 / 隐喻）都服务于内容目标
- 评估：内容目标是否达成？

## 工作流：默认极简 + 复杂扩展

### 默认 · 极简 workflow

**适用**：内容目标明确 + 参考图清晰 + 内容结构清晰

```
内容目标：<读者看完要记住什么/做什么>  [必填]
内容：<原文>                            [必填]
参考图：<图路径>                        [可选]
呈现约束：<如"重点用大字号">             [可选]
```

**直接生图，不拆阶段。** 一句话指令 + 内容就够。

### 扩展 · 4 阶段 workflow（仅复杂任务）

**适用**：内容目标模糊 / 无参考图 / 内容需深度拆解

| 阶段 | 目标 | 核心动作 |
|---|---|---|
| 1 锁定内容目标 | 明确读者看完要什么 | 3 问：内容目标 / 读者 / 参考图 |
| 2 设计呈现策略 | 视觉如何强化内容 | 推导重点/辅助/金句层级 |
| 3 内容-视觉映射 | 每页讲什么 + 怎么呈现 | 拆 N 页 + 每页映射 |
| 4 把映射变成图 | 实际生图 | 组合 prompt + 调 image2 |

**默认走极简。复杂再扩展。**

## 极简 prompt 模板（4 字段）

```
内容目标：<读者看完要记住什么/做什么>
内容：<原文>
参考图：<图>（可选）
呈现约束：<硬约束>（可选）
```

### 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| **内容目标** | ✓ | 读者看完要记住 / 感受 / 做什么 |
| **内容** | ✓ | 原文，直接给 |
| **参考图** | 推荐 | 风格 / 呈现参考（image2 继承） |
| **呈现约束** | 可选 | 只写"内容相关"的硬约束 |

### 禁忌

- ❌ 不要把"内容目标"写成"风格描述"（"治愈水彩风"不是内容目标）
- ❌ 不要写"上下分割 2x2"（让 AI 决定）
- ❌ 不要写比例（用工具参数 `aspect_ratio`）
- ❌ 不要超过 4 字段

### 一图一目标原则（**重要**）

**一个内容目标 = 一个读者动作 / 一个认知 / 一个情感。**

多个内容主题 → 拆成多个 prompt（每图一个），不要硬塞。

| 错误 | 正确 |
|---|---|
| 内容目标 = "让读者掌握 PDP 协议的价值 + 6 种使用方法" | 内容目标 1 = "让读者了解 PDP 协议的价值" / 内容目标 2 = "让读者掌握 PDP 协议的 6 种使用方法" |
| 内容目标 = "介绍产品 + 给出购买链接 + 展示用户评价" | 内容目标 1 = "让读者了解产品" / 内容目标 2 = "让读者下单" / 内容目标 3 = "让读者信任" |

**判定 checklist**：
- [ ] 读者看完有且仅有 1 个动作/认知/情感？
- [ ] 不会出现"既要 X 又要 Y"？
- [ ] 一句话能说清这个目标？

**违反 → 拆分**。

### 完整示例

```
内容目标：让读者掌握 PDP 协议的 6 种使用方法，收藏备用
内容：
[PDP 协议全文 + 6 种使用方法的描述]

参考图：用户提供的 PDP 介绍图
呈现约束：
- 6 种方法各占一个分块
- 重点用大字号
- 整体简洁克制
```

## 扩展 4 阶段详解（仅复杂任务）

### 阶段 1 · 锁定内容目标

**必问 3 问**：
1. 内容目标是什么？（读者看完要记住 / 感受 / 做什么）
2. 读者是谁？
3. 有参考图吗？

**判定 checklist**：
- [ ] 内容目标能用一句话说清
- [ ] 读者画像明确
- [ ] 有 / 无参考图已确定

**产出**：内容目标 + 读者 + 参考图状态

### 阶段 2 · 设计呈现策略

**目标**：视觉如何强化内容？

**动作**：
- 基于内容目标推导：哪些信息要强化 / 弱化
- 推导"重点 / 辅助 / 金句"的层级
- 推导参考图风格 / 新风格

**产出**：呈现策略描述

### 阶段 3 · 内容-视觉映射

**目标**：每页讲什么 + 怎么呈现？

**动作**：
- 拆 N 页
- 每页写：主标题 / 要点 / 视觉策略

**产出**：每页的"内容 + 视觉"映射表

### 阶段 4 · 把映射变成图

**目标**：实际生图

**动作**：
- 组合 prompt（内容目标 + 内容 + 参考图 + 呈现约束）
- 调用 image2
- 内容传达评估

**产出**：每页图

---

## 内容传达评估

| 评估项 | 标准 |
|---|---|
| 核心信息是否突出？ | 读者能否 2 秒 get 重点？ |
| 内容目标是否达成？ | 读者看完是否会做"内容目标"指定的事？ |
| 视觉是否服务于内容？ | 装饰是否掩盖内容？ |
| 是否有错字 / 漏字？ | 文字准确性 |

**不达标 → 调整 prompt 重做，最多 3 次。**

```bash
mavis mcp call apimart-image apimart_generate_image '{
  "prompt": "<完整 prompt 字符串>",
  "aspect_ratio": "3:4",
  "reference_image": "<路径>" (可选)
}'
```

```bash
mavis mcp call apimart-image apimart_get_task '{"task_id": "<id>"}'
```

## 硬约束

- **比例固定**：3:4 竖版（小红书原生）
- **排版不约束**：让 image2 决定
- **必须用 image2 (gpt-image-2)**
- **内容目标必填**：其他字段都围绕它
- **一图一目标**：违反 → 拆分
- **默认极简**：复杂任务再扩展
- **评估以"内容传达"为标准**：不是图规格
